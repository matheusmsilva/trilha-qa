## Plano de Estudos — Automação de Testes com JavaScript, TypeScript e Playwright

## Objetivo da trilha

Esta trilha foi criada para ajudar QAs sem experiência prévia em programação ou automação a desenvolverem uma base sólida em lógica de programação, automação de APIs e automação Web utilizando tecnologias modernas do mercado.

A ideia é começar do absoluto básico e evoluir gradualmente até temas mais avançados, sempre com foco prático e aplicado ao contexto de qualidade de software.

---

# Como a trilha será conduzida

O treinamento seguirá uma abordagem progressiva:

1. Fundamentos de lógica e programação
2. Automação de APIs
3. Fundamentos Web
4. Automação Web com Playwright
5. Estruturação profissional de frameworks
6. Integração com CI/CD
7. Uso de IA no processo de automação

---

# Tecnologias utilizadas

Durante a trilha iremos trabalhar com:

- JavaScript
- TypeScript
- Playwright
- Git
- Node.js
- APIs REST
- Azure Pipelines

---

# Estratégia de aprendizado

O foco principal será:

- prática constante
- pequenos exercícios
- desafios progressivos
- exemplos aplicados ao dia a dia de QA
- resolução de problemas reais

A proposta é formar QAs capazes de:

- entender código
- criar automações
- estruturar testes
- investigar falhas
- colaborar tecnicamente com o time

# Objetivo final

Ao final da trilha, o QA deverá ser capaz de:

- criar automações de API e Web
- estruturar projetos de automação
- trabalhar com Playwright + TypeScript
- executar testes em pipelines
- utilizar IA como apoio no desenvolvimento de automações
- contribuir tecnicamente com estratégias de qualidade

---

# Importante

A evolução será gradual.

Não é necessário possuir experiência prévia com programação para acompanhar o conteúdo.

O mais importante durante a trilha será:

- consistência
- prática
- curiosidade
- experimentação

Automação é uma habilidade construída com repetição e prática contínua.
